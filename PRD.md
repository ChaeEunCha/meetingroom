# PRD: meetingroom (회의실 예약 시스템)

## 1. 개요

| 항목 | 내용 |
|---|---|
| 프로젝트명 | meetingroom |
| 한 줄 설명 | 회의실 예약 현황을 한눈에 보고, 빈 시간을 클릭해 바로 예약할 수 있는 웹사이트 |
| 대상 사용자 | 사내 임직원 (예약자), 총무/관리자 |
| 대상 회의실 | 1번 ~ 6번 회의실 (총 6개) |
| DB | Supabase (PostgreSQL) |

### 배경 / 문제 정의
현재 회의실 예약이 엑셀(`meetingroom_reservations.xlsx`)로 관리되고 있어, 실시간 현황 파악이 어렵고 중복 예약(더블 부킹) 위험이 있다. 회의실별 예약 현황을 시각적으로 보여주고, 빈 시간대를 클릭 한 번으로 예약할 수 있는 웹 서비스를 제공한다.

## 2. 목표

- 6개 회의실의 예약 현황을 하나의 화면에서 한눈에 파악할 수 있다.
- 빈 시간대를 클릭하면 바로 예약 폼이 열리고, 몇 번의 입력만으로 예약이 완료된다.
- 이중 예약(시간 겹침)을 시스템이 자동으로 방지한다.
- 예약 취소/조회가 간단하다.

### Non-Goals (이번 범위에서 제외)
- 로그인/회원가입, 사내 인증(SSO) 연동 — 1차 버전은 예약자 이름/부서 직접 입력 방식으로 진행 (추후 확장 항목 참고)
- 회의실 대여 승인 결재 프로세스
- 회의실 자동 배정(추천) 로직
- 알림(이메일/슬랙) 발송

### 시트 2. 예약현황 (예시 26건)
컬럼: `예약번호`, `회의실번호`, `예약자`, `부서`, `회의제목`, `예약일자`, `시작시각`, `종료시각`, `상태(확정/취소)`

- 예약 시간은 09:00~18:00 사이, 1시간 단위가 기본이나 2시간짜리 예약도 존재 → **시간 슬롯은 30분 또는 1시간 단위 그리드**로 설계하고, 예약은 시작~종료 시각 자유 입력(그리드 스냅) 허용.
- 상태값은 `확정`, `취소` 두 가지 → 취소 건은 화면상 "빈 시간"으로 취급.

## 4. 핵심 기능

### 4.1 예약 현황 대시보드 (메인 화면)
- 6개 회의실 × 시간대(예: 09:00~18:00, 30분 or 1시간 단위) 그리드 뷰
- 세로축: 회의실(1~6번), 가로축: 시간 (또는 반대 배치)
- 날짜 선택(날짜 피커, 기본값: 오늘) → 해당 일자의 예약 현황 표시
- 예약된 슬롯: 색상 채움 + 예약자/회의제목 표시 (마우스 오버 시 상세 툴팁: 예약자, 부서, 시간, 제목)
- 빈 슬롯: 클릭 가능한 상태로 표시 (hover 시 "예약하기" 커서/하이라이트)
- 취소된 예약은 목록에서 제외(빈 시간으로 노출)

### 4.2 빠른 예약(퀵 북킹)
- 빈 슬롯 클릭 → 예약 모달 오픈 (회의실/날짜/시작시각 자동 입력됨)
- 입력 항목: 예약자명(필수), 부서(필수), 회의 제목(필수), 종료시각(드래그로 슬롯 여러 개 선택 시 자동 계산, 기본 1시간)
- 종료 시각은 드롭다운 또는 슬롯 드래그로 조정 가능
- "예약 확정" 클릭 시 즉시 DB 반영 → 대시보드 실시간 갱신
- 서버(DB) 단에서 동일 회의실·시간대 중복 예약 검증 → 겹치면 에러 메시지 노출

### 4.3 예약 목록 / 상세 조회
- 회의실별, 날짜별, 예약자별 필터가 가능한 예약 리스트 화면
- 예약 상세에서 취소 가능 (상태를 `취소`로 변경, soft delete)

### 4.4 예약 취소
- 대시보드에서 본인 예약 클릭 → 상세 팝오버에서 "취소" 버튼
- 취소 시 확인 다이얼로그 노출 후 상태 변경

### 4.5 회의실 정보
- 회의실 카드/툴팁에 수용인원, 층, 보유 장비, 비고 표시 (예약 시 참고 가능)

## 5. 사용자 시나리오

1. **회의실 예약** — 사용자가 대시보드에서 오늘 날짜 그리드를 보고, 3번 회의실 14:00~15:00 빈 슬롯을 클릭 → 이름/부서/제목 입력 후 확정 → 그리드에 즉시 반영.
2. **현황 확인** — 임원 보고 전, 5번 회의실(대회의실)이 비어있는 시간을 빠르게 확인.
3. **예약 취소** — 회의가 취소되어 본인이 잡은 예약을 클릭해 취소 처리, 해당 슬롯이 다시 빈 시간으로 노출.
4. **중복 방지** — 동시에 두 사용자가 같은 슬롯을 클릭할 경우, 나중에 확정을 시도한 사용자에게 "이미 예약된 시간입니다" 에러 표시.

## 6. 데이터 모델 (Supabase / PostgreSQL)

### `rooms` 테이블
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | smallint (PK) | 회의실번호 (1~6) |
| name | text | 회의실명 (예: "1번 회의실 (소회의실 A)") |
| capacity | smallint | 수용인원 |
| floor | text | 층 (예: "3층") |
| equipment | text | 보유장비 (콤마 구분 문자열 또는 text[]) |
| note | text (nullable) | 비고 |

### `reservations` 테이블
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid (PK, default `gen_random_uuid()`) | 내부 식별자 |
| reservation_code | text (unique, nullable) | 사람이 읽는 예약번호 (예: B2026071), 자동 생성 |
| room_id | smallint (FK → rooms.id) | 예약 회의실 |
| reservee_name | text | 예약자명 |
| department | text | 부서 |
| title | text | 회의 제목 |
| reservation_date | date | 예약일자 |
| start_time | time | 시작시각 |
| end_time | time | 종료시각 |
| status | text (`confirmed` \| `cancelled`, default `confirmed`) | 상태 |
| created_at | timestamptz (default now()) | 생성일시 |
| updated_at | timestamptz (default now()) | 수정일시 |

### 제약조건 / 정합성
- `end_time > start_time` CHECK 제약
- 동일 `room_id` + `reservation_date`에서 `status = 'confirmed'`인 예약끼리 시간 겹침 금지
  - Postgres `EXCLUDE USING gist` 제약 (예: `tsrange` 변환 후 exclusion constraint) 또는 애플리케이션 레벨 트랜잭션 검증으로 구현
- Supabase Realtime을 이용해 `reservations` 테이블 변경을 구독 → 대시보드 실시간 반영

## 7. 화면 설계 (초안)

1. **메인 대시보드**: 상단 날짜 선택 + 회의실 필터, 중앙에 회의실(행) × 시간(열) 그리드
2. **예약 모달**: 회의실명/날짜/시간 표시(수정 가능) + 예약자명/부서/제목 입력 폼 + 확정 버튼
3. **예약 목록 페이지**: 테이블 형태, 검색/필터(날짜, 회의실, 예약자, 상태)
4. **회의실 정보 패널**: 그리드 상단 또는 사이드에 회의실별 수용인원/장비 요약 카드

## 8. 비기능 요구사항

- **실시간성**: 예약/취소 시 다른 사용자 화면에도 수 초 내 반영 (Supabase Realtime 구독)
- **동시성**: 동시 예약 시도 시 DB 제약조건으로 최종 정합성 보장
- **반응형**: 데스크톱 우선, 태블릿에서도 그리드 확인 가능한 반응형 레이아웃
- **성능**: 하루 기준 6개 회의실 × 예약 데이터 조회는 단일 쿼리로 충분히 빠르게 처리

## 9. 기술 스택 (제안)

- Frontend: Next.js (React) + Tailwind CSS
- Backend/DB: Supabase (PostgreSQL, Auth 추후 확장, Realtime 구독)
- 배포: Vercel

## 10. 향후 확장 (Out of Scope, 2차 이후)

- Supabase Auth 연동 (사내 계정 로그인, 본인 예약만 취소 가능하도록 권한 제어)
- 이메일/슬랙 알림 (예약 확정, 임박 알림, 취소 알림)
- 반복 예약(매주 회의 등)
- 회의실 이용 통계 대시보드 (회의실별 이용률, 부서별 이용 현황)
- 모바일 앱 / PWA
